ATICA OPS — MODULAR SPLIT GAME PLAN

How to break the 1MB monolith into modules that work on the same system and don't step on each other

THE PROBLEM RIGHT NOW

  • atica_app.html is 1,030,873 bytes — everything in one file. Every edit risks breaking someone else's work.
  • shopify.js function is a god-function — products, orders, inventory, ledger, velocity, POS all in one 230-line handler.
  • No module boundaries — Shrek touching MPs can break Deshawn's cash flow, Stallon's API changes can break the frontend.
  • localStorage only — no shared state, no persistence beyond the browser.

GOLDEN RULES — DON'T BREAK EACH OTHER

  • Own your file, only your file. Each module is a separate JS file. You edit yours, nobody else's.
  • Communicate through the event bus. Never reach into another module's DOM or state directly.
  • Shared data goes through lib/. The Shopify client, mappers, and types are shared — changes need everyone's sign-off.
  • Every function gets its own Netlify function file. No more 200-line god-function.
  • Test your endpoint before pushing. If your function breaks, it breaks for everyone.

HOW MODULES TALK TO EACH OTHER

Event Bus Pattern

Every module imports event-bus.js. Nobody reaches into another module's state. You publish events, others subscribe.

// event-bus.js — shared communication layer
const bus = {};
export function emit(event, data) {
  (bus[event] || []).forEach(fn => fn(data));
}
export function on(event, fn) {
  (bus[event] ||= []).push(fn);
}

// In pos.js (Deshawn publishes)
bus.emit('order:new', orderData);

// In cash-flow.js (Deshawn subscribes)
bus.on('order:new', (order) => {
  updateRevenueStream(order);
});

// In marketplace.js (Shrek subscribes)
bus.on('order:new', (order) => {
  updateSellThrough(order);
});

MODULE TEMPLATE — EVERY MODULE FOLLOWS THIS PATTERN

// modules/[your-module].js
import { on, emit } from './event-bus.js';
import { api, render } from './core.js';

// ── State (private to this module) ──
let state = { loaded: false, data: null };

// ── Init (called once when module mounts) ──
export async function init(container) {
  container.innerHTML = renderSkeleton();
  state.data = await api.get('/api/your-endpoint');
  state.loaded = true;
  update(container);
}

// ── Update (re-render with current state) ──
function update(container) {
  container.innerHTML = renderView(state);
  bindEvents(container);
}

// ── Listen to other modules ──
on('order:new', (order) => {
  // React to events from other modules
  state.data.push(transform(order));
  update(document.getElementById('content'));
});

// ── Publish to other modules ──
function onAction(item) {
  emit('your-module:action', item);
}
