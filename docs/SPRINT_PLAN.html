Remediation Sprint Plan

Atica OPS — What to Fix, When

22 tasks across 6 sprints. Ordered by blast radius and dependency chain. Constraint: Shopify remains source of truth for now. All fixes work within the current Next.js + Neon + Netlify stack.
Operating Constraint

Shopify stays as source of truth. The sync-from-Shopify architecture remains. All fixes below are designed to work with the current data flow direction (Shopify → Sync → Postgres). Flipping to ERP-as-source-of-truth is a Phase 2 project for Q3/Q4 2026.

This means: we improve how we receive, store, and use Shopify data — but we don't try to push products TO Shopify yet. Webhooks supplement the sync. The SKU table derives from Shopify variants. Inventory events layer on top of Shopify's inventory data.

CREATE TABLE skus (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  style_id UUID NOT NULL REFERENCES styles(id) ON DELETE CASCADE,
  fit TEXT,               -- 'Lorenzo', 'Alexander', 'Slim', NULL
  size TEXT NOT NULL,     -- '38', '40', '15.5', 'M'
  length TEXT,            -- 'S', 'R', 'L', NULL for shirts
  sku_code TEXT UNIQUE,   -- computed: style_number + fit + size + length
  barcode TEXT,
  shopify_variant_id BIGINT UNIQUE,
  shopify_inventory_item_id BIGINT,
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(style_id, fit, size, COALESCE(length, ''))
);
CREATE INDEX idx_skus_style ON skus(style_id);
CREATE INDEX idx_skus_shopify_variant ON skus(shopify_variant_id);

Sprint 2 Event-Sourced Inventory Week 3-4 — 5-7 days

Replace the empty store_inventory table with an event-sourced model. Every inventory change becomes an auditable event. Current stock = SUM of events. This is the foundation for transfers, receiving, and multi-store visibility.

Create inventory_events table + materialized view
Almond
3 hrs
›

The core table:

CREATE TABLE inventory_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sku_id UUID NOT NULL REFERENCES skus(id),
  location_id UUID NOT NULL REFERENCES locations(id),
  event_type TEXT NOT NULL,
  quantity INT NOT NULL,  -- positive=add, negative=remove
  reference_type TEXT,    -- 'shopify_sync','sale','po_receipt',
                          -- 'transfer','adjustment','return'
  reference_id UUID,
  notes TEXT,
  created_by TEXT DEFAULT 'system',
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_inv_events_sku ON inventory_events(sku_id);
CREATE INDEX idx_inv_events_loc ON inventory_events(sku_id, location_id);

-- Current stock = SUM of all events
CREATE MATERIALIZED VIEW inventory_levels AS
SELECT sku_id, location_id,
  SUM(quantity) AS on_hand,
  MAX(created_at) AS last_movement
FROM inventory_events
GROUP BY sku_id, location_id
HAVING SUM(quantity) != 0;

CREATE UNIQUE INDEX idx_inv_levels
  ON inventory_levels(sku_id, location_id);

Why materialized view: REFRESH MATERIALIZED VIEW CONCURRENTLY gives you instant reads without computing SUM on every query. Refresh after sync runs and after webhook batches. Between refreshes, use a get_stock(sku_id, location_id) function for real-time reads.

3 hrs
›
